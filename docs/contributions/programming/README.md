# Programming Contributions

The project is currently in its early stages, and the internal data structure is still evolving. While it is functional, the way objects are defined and stored is currently a bit *weird*.

I am actively looking for contributors to help:

1. Review how structural members (beams, columns, etc.) are currently handled in the Ruby API.
2. Suggest more efficient ways to structure the metadata or object relationships to make the extension more "BIM-like"
3. Help transition the current *weird* logic into a more scalable, standardized architecture.
4. Develop a test package (currently there aren't any).

If you have experience with Design Patterns, I would love to hear your thoughts! You can open an Issue first so we can 
coordinate our efforts or message me on [Twitter](https://x.com/structureline31).
